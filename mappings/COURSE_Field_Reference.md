
# COURSE Table — Field Reference (Authoritative)

## Identifiers & Keys

- **ID** — Snowflake-generated surrogate key; not sourced from Blackboard.
- **INSTANCE_ID** — Identifier for the Blackboard deployment; all records from a single deployment share this key.
- **SOURCE_ID** — Blackboard database primary key (`pk1`). Primary keys are not reused in Blackboard. Value may be observed in course URLs.
- **COURSE_NUMBER** — Blackboard visible Course ID. Unique in Blackboard, not guaranteed unique in Snowflake due to historical retention.
- **STAGE:uuid** — Key value generated by Blackboard used for integrating with other systems (LTI).
- **STAGE:batch_uid** — Blackboard external course key; required to be unique in Blackboard. Typically sourced from a system of record (e.g., SIS).

## Relationships

- **COURSE_PARENT_ID** — Lookup to `COURSE.id` of the parent (merged) course.
- **TERM_ID** — Lookup to `TERM.id` value in Snowflake.
- **COPY_FROM_COURSES** — Lookup to `COURSE.id` values for source courses from which copies were made.

## Descriptive Metadata

- **NAME** — Blackboard Course Name.
- **DESCRIPTION** — Blackboard Course Description.

## Course Experience / Design

- **DESIGN_MODE** — Blackboard Course Experience (Ultra or Original).
- **DESIGN_MODE_SOURCE_CODE** — Duplicate of `DESIGN_MODE`.
- **DESIGN_MODE_SOURCE_DESC** — Not implemented.

## Enrollment Configuration

- **ENROLLMENT_METHOD** — Blackboard enrollment setting: `I` = Instructor/Admin only; `E` = email approval; `S` = self-enrollment.
- **ENROLLMENT_METHOD_SOURCE_CODE** — Duplicate of `ENROLLMENT_METHOD`.
- **ENROLLMENT_METHOD_SOURCE_DESC** — Not implemented.

## Availability & Enablement

- **AVAILABLE_IND** — Direct value from the Blackboard course record indicating availability.
- **AVAILABLE_TO_STUDENTS_IND** — Derived value based on availability rules applied to the course.
- **ENABLED_IND** — Direct value from the Blackboard course record indicating system enablement.
- **STAGE:avl_rule_indicator** — Indicates whether availability rules are applied.

## Scheduling (Configured)

- **START_DATE** — Date portion of the course start date in Blackboard.
- **END_DATE** — Date portion of the course end date in Blackboard.
- **START_TIME** — Full timestamp of the course start date in Blackboard.
- **END_TIME** — Full timestamp of the course end date in Blackboard.

## Access Activity (Derived)

- **FIRST_COURSE_TIME** — Timestamp of first known access by any user, derived from `CDM_LMS.course_activity`.
- **LAST_COURSE_TIME** — Timestamp of last known access by any user, derived from `CDM_LMS.course_activity`.
- **FIRST_COURSE_WEEK** — Monday preceding `FIRST_COURSE_TIME`.
- **LAST_COURSE_WEEK** — Monday preceding `LAST_COURSE_TIME`.

## Lifecycle & Audit

- **CREATED_TIME** — Timestamp when the course was created in Blackboard.
- **MODIFIED_TIME** — Timestamp when the course was last updated in Blackboard.
- **ROW_INSERTED_TIME** — Timestamp when the record was created in Snowflake.
- **ROW_UPDATED_TIME** — Timestamp when the record was updated in Snowflake.
- **ROW_DELETED_TIME** — Timestamp when the Snowflake record was marked deleted after Blackboard deletion.

## Data Source & Governance (STAGE)

- **STAGE:data_src_pk1** — Internal primary key of the course’s data source in Blackboard.
- **STAGE:data_src_batchuid** — Visible identifier of the data source in Blackboard.
- **STAGE:service_level** — Course type specialization: `F` = Course, `C` = Organization, `O` = LOR, `J` = Subject, `P` = Program.
- **STAGE:allow_guest** — Guest access setting (`Y`/`N`).
- **STAGE:allow_observer** — Observer access setting (`Y`/`N`).
- **STAGE:sos_id_pk2** — Deprecated; former tenant key for virtual deployments.
- **STAGE:foundations_id** — Internal Blackboard identifier for integration with other Blackboard services.
